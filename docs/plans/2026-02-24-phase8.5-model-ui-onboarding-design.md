# Phase 8.5 Design: Model Selection + UI Polish + Onboarding

**Date**: 2026-02-24
**Status**: Approved
**Scope**: 3 batches — Model Selection → UI Polish → Onboarding Redo

---

## Context

VoxInk v1.0 is code-complete with core IME, Groq STT, LLM refinement, transcription, and monetization. User testing reveals three areas needing improvement:

1. **No model choice** — hardcoded `whisper-large-v3-turbo` for STT and `llama-3.3-70b-versatile` for LLM. Users should pick models.
2. **UI feels basic** — no recording animation, no haptic feedback, generic system icons, plain Home Screen.
3. **Onboarding lacks practice** — 5-step text-only flow without hands-on demo or button explanations.

### Competitive Research

**Typeless**: "Speak to Edit" button, calibration phrases in onboarding, extreme minimalism.
**Wispr Flow**: Non-skippable onboarding with practice dictation, role-based personalization, context-aware formatting. Behavioral scientist called it "the best onboarding I've ever seen."

Both apps invest heavily in recording feedback (pulse animation, haptic) and post-transcription editing (deferred to future for VoxInk).

## Batch 1: Model Selection

### Goal

Let users choose STT and LLM models within the same Groq API key. No new provider keys needed.

### STT Model Selector

Add a "STT Model" section in SettingsScreen between "Language" and "Recording Mode":

- **`whisper-large-v3-turbo`** (default) — label: "Turbo — 更快、更便宜 (US$0.04/hr)"
- **`whisper-large-v3`** — label: "V3 — 更精確, WER 8.4% (US$0.111/hr)"

Description text below selector:
> Turbo 便宜約 2.8 倍，日常語音輸入推薦使用。V3 精確度較高，適合重要或專業場合。兩者都由 Groq 提供。
>
> Turbo is ~2.8x cheaper and recommended for daily use. V3 has higher accuracy (8.4% WER) for critical work. Both powered by Groq.

### LLM Model Selector

Add a "LLM Model" section in SettingsScreen after "STT Model":

- **`llama-3.3-70b-versatile`** (default) — label: "LLaMA 3.3 70B — 平衡速度與品質"
- **`gpt-oss-120b`** — label: "GPT-OSS 120B — OpenAI 開源, ≈ o4-mini"
- **`gpt-oss-20b`** — label: "GPT-OSS 20B — 輕量快速, ≈ o3-mini"

Description text:
> GPT-OSS 是 OpenAI 的開源模型（Apache 2.0），可透過 Groq API 使用，無需額外 key。120B 效能接近 o4-mini，20B 更輕更快。
>
> GPT-OSS are OpenAI's open-weight models (Apache 2.0), available via Groq API with no extra key. 120B rivals o4-mini; 20B is lighter and faster.

### Data Layer Changes

- `PreferencesManager`: add `sttModel: Flow<String>` and `llmModel: Flow<String>` (DataStore keys)
- `SttRepository.transcribe()`: read `sttModel` preference, pass as `model` parameter to Groq API
- `LlmRepository.refine()`: read `llmModel` preference, pass as `model` parameter to chat completion
- `SettingsViewModel`: expose model selections as StateFlow, handle updates
- `strings.xml` / `strings.xml (zh-rTW)`: add bilingual labels and descriptions

### Tests

- `PreferencesManagerTest`: verify default values and persistence for new keys
- `SttRepositoryTest`: verify correct model sent in API request
- `LlmRepositoryTest`: verify correct model sent in API request
- `SettingsViewModelTest`: verify model selection state management

## Batch 2: UI Polish

### Goal

Make the app feel polished — recording feedback, visual hierarchy, Home Screen redesign.

### 2A: Recording Feedback (IME)

**Mic Pulse Animation:**
- When recording, mic button background pulses: scale 1.0 → 1.15 → 1.0, alpha 1.0 → 0.7 → 1.0
- Use Android `ObjectAnimator` on the ImageButton (XML-based keyboard, not Compose)
- Duration: 800ms per cycle, repeat infinite
- Color: existing `MicActive` red (#EF4444)

**Haptic Feedback:**
- Recording start: `HapticFeedbackConstants.LONG_PRESS` via `View.performHapticFeedback()`
- Recording stop: `HapticFeedbackConstants.CONFIRM` (API 30+), fallback to `KEYBOARD_TAP`
- Transcription complete (text committed): `CONFIRM`

**Recording Timer:**
- Display elapsed time in candidate status row: "Recording 0:03" format
- Use `Handler.postDelayed()` with 1-second interval, update status text
- Timer resets on stop

**Sound Effects:**
- Start recording: short ascending tone via `ToneGenerator(TONE_PROP_BEEP)`
- Stop recording: short descending tone
- Keep volume low, respect system media volume

### 2B: Candidate Bar Improvements (IME)

**Visual Distinction:**
- Original row: prefix with "○" circle icon, text color `#9E9E9E` (dimmer gray), font size 13sp
- Refined row: prefix with "✦" icon in `VoxInkPurple`, text color `#FFFFFF`, font size 15sp, slightly bold
- Add 1dp divider between original and refined rows

**Transition Animation:**
- Candidate bar appearance: slide up from bottom + fade in (200ms)
- State changes (Recording → Processing → Result): cross-fade text (150ms)
- Use Android `TransitionManager` or `ValueAnimator` for XML views

### 2C: Icon Replacement (IME)

Replace system drawables with Material-style vector drawables:
- `ic_mic_idle` → custom mic icon in VoxInkPurple
- `ic_mic_active` → custom mic icon in red with recording indicator dot
- `ic_backspace` → Material `backspace` rounded
- `ic_enter` → Material `keyboard_return` rounded
- `ic_settings` → Material `settings` rounded
- `ic_switch_keyboard` → Material `language` rounded

### 2D: Home Screen Redesign

**Setup Checklist Card:**
- Each item is a Row with: status icon (✅ green check or ⚠️ orange warning) + label + current value
- Items are clickable — navigate to the relevant setup action:
  - "Keyboard" → open system IME settings
  - "API Key" → navigate to Settings
  - "Microphone" → request permission
- Card uses `ElevatedCard` with rounded corners

**Usage Summary (Free users):**
- 3 horizontal `LinearProgressIndicator` bars:
  - Voice Input: `X / 20` with progress color gradient (green → yellow → red as approaching limit)
  - Refinement: `X / 5`
  - File Transcription: `X / 2`
- Each bar has label + count text
- Reset time indicator: "重置於午夜 / Resets at midnight"

**Overall Layout:**
- Top: App logo + tagline in a hero-style header
- Middle: Setup checklist card (dismissible once all complete)
- Below: Usage summary card (Free only)
- Action buttons: `FilledTonalButton` for primary actions, better spacing
- Bottom: Banner ad (Free only)

### Tests

- `HomeScreenTest`: verify checklist renders with correct icons, usage bars show correct progress
- No unit tests needed for animations (visual verification)
- `RecordingControllerTest`: verify timer updates during recording state

## Batch 3: Onboarding Redo

### Goal

Guide new users through setup with practice dictation and visual polish. Non-skippable.

### Revised Flow (6 steps)

**Step 1: Welcome**
- App logo centered
- "VoxInk — AI 語音鍵盤" / "VoxInk — AI Voice Keyboard"
- One-line description: "用說的，比打字更快" / "Speak instead of type"
- Single "開始 / Get Started" button
- No back button, no skip

**Step 2: API Key**
- Text field for Groq API key
- "從 console.groq.com 取得" clickable link (opens browser)
- On save: test API call to validate key (send tiny audio clip or list models endpoint)
- Success: green checkmark animation
- Failure: error message with retry

**Step 3: Enable Keyboard**
- Illustration/icon showing system keyboard settings
- "開啟設定" button → opens Android IME settings intent
- Auto-detect when keyboard is enabled (poll `InputMethodManager.getEnabledInputMethodList()`)
- Show ✅ when detected

**Step 4: Microphone Permission**
- Explain why: "VoxInk 需要麥克風來錄製語音" / "VoxInk needs microphone access to record speech"
- Standard Android permission request
- If denied: explain how to enable in system settings

**Step 5: Practice Recording** ⭐ NEW
- Show a mock text input area within the onboarding screen
- Instruction: "試試看！按下麥克風按鈕，說一句話" / "Try it! Tap the mic button and say something"
- Embedded mic button (same style as keyboard)
- After recording: show Original → Refined result inline
- Celebratory feedback: "太棒了！" / "Great!" with confetti or checkmark
- User can retry or proceed

**Step 6: Done**
- "準備好了！" / "You're all set!"
- Summary of what was configured
- "開始使用" / "Start Using VoxInk" button → navigate to Home Screen

### Progress Indicator

Replace numbered steps (1/6, 2/6) with contextual text:
- Steps 1-2: no progress text (just the content)
- Step 3: "還有幾步..." / "A few more steps..."
- Step 4: "快完成了" / "Almost there"
- Step 5: "最後一步！" / "Last step!"
- Step 6: (no progress text, just celebration)

Keep the thin linear progress bar at top for subtle visual cue, but remove the "Step X of Y" counter.

### Non-Skippable

- Remove any skip buttons
- Back button only goes to previous step, not skip ahead
- Each step requires completion before Next is enabled:
  - Step 2: API key saved
  - Step 3: keyboard enabled
  - Step 4: permission granted
  - Step 5: at least one practice recording completed

### Keyboard Tooltips (First Launch)

Separate from onboarding — triggered the first time the VoxInk keyboard appears in any app:

- Semi-transparent dark overlay on keyboard
- Each button has a tooltip bubble above it:
  - 🎤 "錄音 / Record"
  - ⌫ "刪除 / Delete"
  - 🌐 "切換鍵盤 / Switch Keyboard"
  - ⏎ "換行 / Enter"
  - ⚙️ "設定 / Settings"
- Tap anywhere to dismiss
- `DataStore` boolean `hasSeenKeyboardTooltips` — show only once
- Implementation: overlay `FrameLayout` with tooltip views, absolute positioning relative to each key

### Tests

- `OnboardingViewModelTest`: verify step progression, non-skippable enforcement, practice recording state
- `OnboardingScreenTest`: verify UI renders for each step, API key validation feedback
- Tooltip: manual verification (visual overlay)
